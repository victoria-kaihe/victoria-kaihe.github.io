Welcome back, non-tech friend. It's time for Tori to explain you a thing that she has only recently learned herself. Today we're talking JavaScript function scope. There are two kinds of scope - global, and local. If a variable is declared outside of a function, it has global scope. This means that it can be accessed anywhere else in the JavaScript document. You could think of it like an unclassified document - anyone can access and read it. Variables created within a function are local, and can only be accessed by that function - or another function within that function. You could think of local scope like higher levels of security cleareance. Say you create a variable within a function, and it has local scope that we'll think of as Confidential. Then within that function, you create another function, with its own variables, but these are more local than Confidential - these are Secret. So the Secret level function has access to the Confidential variables, but Confidential can't see or affect what's happening at Secret level - it doesn't have the right clearance (or scope). And then of course it's all need-to-know - so you might have a block of nested functions and they're doing their job, and another block of functions doing a completely different job, so they don't interact with each other or share their variables - they probably don't need to.
