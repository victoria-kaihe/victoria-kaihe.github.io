Responsive Design Meditation Process

A responsive site is a website which responds differently to different-sized windows or displays. The layout will adjust depending on what sized screen or device it is being displayed on.

Mobile-first design is starting the design process with the mobile version of a site. Because of the size of mobile phone screens, space is a premium. This allows the person designing the page to focus just on what elements are the most important. From there, more elements and more design features can be added as larger screen sizes allow.

A framework is a package of standardized code documents that can be used as a foundation or template on which to build a website. For example, this website has been recently rebuilt using the Skeleton framework.

There are pros and cons to using a framework. Using a framework speeds up the mock-up process, provides clean and tidy code, can be helpful in collaborative work since everyone is working from the same protocol, and browser compatbility is already included. However it can slow the learning curve for new developers, as you're not learning to do everything yourself.

A wireframe is a very simple drawing of your website. It doesn't incorporate any real design features, just the basic elements of the page and where you want them. We use the wireframe as a reference point, a blueprint. It helps us to establish a hierarchy of all our elements and the information therein.
